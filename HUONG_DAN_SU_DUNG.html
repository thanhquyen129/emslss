Hướng dẫn sử dụng EMS-LSS

Tài liệu mô tả chức năng đã triển khai: tiếp nhận đơn từ EMS, điều phối pickup / delivery, tracking, callback và các màn hình quản trị — cập nhật theo mã nguồn hiện tại của dự án.

Mục lục 1. Tổng quan 2. Đăng nhập và vai trò 3. Luồng nghiệp vụ 4. Admin / Dispatcher 5. Operation (Hub) 6. Shipper 7. API tích hợp EMS 8. Trạng thái đơn (tham khảo) 9. Ghi chú vận hành

1. Tổng quan

EMS-LSS là hệ thống trung gian: EMS gửi đơn vào LSS, đội vận hành phân công shipper nhận thư (pickup) và phát thư (delivery), ghi nhận tracking và gửi trạng thái về EMS (callback).

EMS → API nhận đơn → CSDL LSS → Dispatcher / Admin phân công → Shipper pickup (scan, ảnh) → Operation / Hub (nhận hàng, giao cho delivery) → Shipper delivery (scan, ảnh) → Callback trạng thái → EMS

Điểm vào ứng dụng web: index.php chuyển hướng tới modules/login.php.

2. Đăng nhập và vai trò

Đăng nhập tại /modules/login.php. Sau khi đăng nhập thành công:

  • shipper: chuyển tới dashboard shipper (modules/shipper/shipper_dashboard.php).
  • Các vai trò khác (admin, dispatcher, operation…): chuyển tới modules/admin/dashboard.php, file này tự điều hướng theo role trong session:
    • admin, dispatcher → realtime dashboard admin.
    • operation → dashboard vận hành (hub).
    • shipper (nếu vào qua đường này) → pickup_dashboard.php (legacy).

Đăng xuất: /logout.php.

3. Luồng nghiệp vụ đã có trong hệ thống

Luồng lõi đã được mã hóa qua các endpoint và module sau (thứ tự logic):

  1. EMS đẩy đơn → API nhận đơn (xem mục API).
  2. Đơn ở trạng thái mới → dispatcher/admin gán shipper pickup (và sau đó gán delivery khi tới bước đó).
  3. Shipper pickup: xem đơn, scan mã EMS, hoàn tất pickup (ảnh chứng từ nếu có).
  4. Operation: nhận hàng từ pickup, có thể gán giao hàng (delivery).
  5. Shipper delivery: scan và hoàn tất giao.
  6. Callback: hệ thống gửi trạng thái về EMS; có cron / màn hình theo dõi callback.

4. Admin / Dispatcher

Các URL dưới đây giả định gốc site là /; nếu cài trong thư mục con, thêm prefix tương ứng.

Chức năng Đường dẫn (canonical)
Dashboard realtime (tổng quan đơn) /modules/admin/admin_dashboard_realtime.php
Điểm vào dashboard (redirect theo role) /modules/admin/dashboard.php
Chi tiết đơn (admin) /modules/admin/admin_order_detail.php?id={order_id}
Quản lý người dùng /modules/admin/admin_users.php
Sửa / tạo user (nếu có) /modules/admin/admin_user_edit.php
Vô hiệu hóa user /modules/admin/admin_user_disable.php
Quản lý vai trò (roles) /modules/admin/admin_roles.php
Báo cáo /modules/admin/admin_reports.php
Theo dõi callback / retry /modules/admin/callback_monitor.php
Nhật ký API /modules/admin/api_logs.php
Gán đơn cho user / shipper (module hỗ trợ) /modules/admin/assign_order_user.php

Cách dùng ngắn gọn

  • Vào realtime dashboard để xem danh sách đơn và trạng thái cập nhật.
  • Mở chi tiết đơn để xem timeline, thông tin gán shipper, ảnh và log liên quan (tùy dữ liệu đã ghi).
  • Callback monitor và API logs dùng khi cần xử lý lỗi tích hợp hoặc tra cứu request/response.
  • Users / Roles để tạo tài khoản, phân quyền phù hợp vận hành.

5. Operation (Hub)

Chức năng Đường dẫn
Dashboard vận hành /modules/operation/dashboard.php
Nhận hàng từ pickup (theo đơn) /modules/operation/receive.php?id={order_id}
Gán delivery (POST API nội bộ) /modules/operation/assign_delivery.php

Có thêm các file bọc legacy như operation_dashboard.php, operation_receive.php tại thư mục modules/ — ưu tiên dùng đường dẫn trong bảng trên khi tích hợp mới.

6. Shipper

Hai nhóm: pickup và delivery. Giao diện shipper tập trung dưới modules/shipper/.

Chức năng Đường dẫn
Dashboard shipper (tổng) /modules/shipper/shipper_dashboard.php
Chi tiết đơn (pickup) /modules/shipper/shipper_order_detail.php?id={order_id}
Scan pickup /modules/shipper/shipper_scan.php?id={order_id}
Hoàn tất pickup /modules/shipper/shipper_complete.php?id={order_id}
Dashboard delivery /modules/shipper/delivery_dashboard.php
Chi tiết delivery /modules/shipper/delivery_detail.php?id={order_id}
Scan giao hàng /modules/shipper/delivery_scan.php?id={order_id}
Hoàn tất giao hàng /modules/shipper/delivery_complete.php?id={order_id}

Module pickup / delivery cũ (vẫn có trong repo)

Các file như pickup_dashboard.php, pickup_scan.php, modules/pickup/, modules/delivery/ có thể song song với luồng shipper — khi vận hành nên thống nhất một bộ URL để tránh nhầm lẫn.

7. API tích hợp EMS

Chi tiết request/response xem API.md trong repo. Các endpoint chính đã có trong mã nguồn:

Mục đích Endpoint
EMS đẩy đơn vào LSS POST /api/ems_push_order.php
EMS tra cứu tracking GET /api/ems_tracking.php?ems_code=...
Hủy đơn (nếu được bật cấu hình) /api/ems_cancel_order.php
Callback nội bộ / pickup / delivery /api/callback_ems.php, /api/callback_pickup.php, /api/callback_delivery.php
Retry callback (hỗ trợ vận hành) /api/callback_retry.php
Health check /api/health.php

Xác thực API thường qua header X-API-KEY (xem API.md và config/).

8. Trạng thái đơn (tham khảo)

Theo tài liệu nghiệp vụ (readme.md), chuỗi tracking có thể gồm các bước như: new_order → assigned_pickup → picked_up → arrived_hub / in_transit → assigned_delivery → delivered (và failed khi lỗi).

Tên trạng thái cụ thể trong CSDL có thể khác một phần theo phiên bản code — khi báo cáo hoặc tích hợp, nên đối chiếu bảng emslss_orders và emslss_tracking trong DB_SCHEMA.md hoặc install.sql.

9. Ghi chú vận hành

  • Cron / retry: thư mục cron/ chứa script retry callback — cần cấu hình scheduler trên server production.
  • Test đẩy đơn: test_ems_push.php dùng kiểm tra tích hợp nhận đơn.
  • Phân biệt chữ hoa/thường: trên Linux, đường dẫn file phân biệt hoa thường (ví dụ api_logs.php vs API_logs.php).
  • Roadmap / hạn chế: đánh giá chi tiết tính năng còn thiếu và đề xuất nâng cấp nằm trong hoan_thien.md.

Tài liệu này mô tả các thành phần đã có trong repository EMS-LSS. Cập nhật file HTML này khi thêm route hoặc thay đổi luồng nghiệp vụ.
