1. Tổng quan

EMS-LSS là hệ thống trung gian: EMS gửi đơn vào LSS, đội vận hành phân công shipper nhận thư (pickup) và phát thư (delivery), ghi nhận tracking và gửi trạng thái về EMS (callback). 

EMS → API nhận đơn → CSDL LSS
→ Dispatcher / Admin phân công
→ Shipper pickup (scan, ảnh)
→ Operation / Hub (nhận hàng, giao cho delivery)
→ Shipper delivery (scan, ảnh)
→ Callback trạng thái → EMS

Điểm vào web: index.php chuyển hướng tới modules/login.php.

2. Đăng nhập và vai trò

Đăng nhập tại /modules/login.php. Sau khi đăng nhập thành công:

  • shipper → modules/shipper/shipper_dashboard.php
  • Các vai trò khác → modules/admin/dashboard.php (tự redirect theo role):
    • admin, dispatcher → dashboard realtime
    • operation → dashboard hub
    • shipper (nếu vào qua admin entry) → pickup_dashboard.php (legacy)

Đăng xuất: /logout.php

3. Luồng nghiệp vụ

  1. EMS đẩy đơn → API nhận đơn (mục API).
  2. Đơn mới → dispatcher/admin gán shipper pickup (sau đó gán delivery khi tới bước đó).
  3. Shipper pickup: xem đơn, scan mã EMS, hoàn tất pickup (ảnh chứng từ nếu có).
  4. Operation: nhận hàng từ pickup, gán giao hàng.
  5. Shipper delivery: scan và hoàn tất giao.
  6. Callback về EMS; cron / màn hình theo dõi callback.

4. Admin / Dispatcher

URL giả định gốc site là /; nếu cài trong thư mục con, thêm prefix tương ứng.

Chức năng Đường dẫn (canonical)
Dashboard realtime/modules/admin/admin_dashboard_realtime.php
Điểm vào (redirect theo role)/modules/admin/dashboard.php
Chi tiết đơn/modules/admin/admin_order_detail.php?id={order_id}
Quản lý người dùng/modules/admin/admin_users.php
Sửa / tạo user/modules/admin/admin_user_edit.php
Vô hiệu hóa user/modules/admin/admin_user_disable.php
Quản lý roles/modules/admin/admin_roles.php
Báo cáo/modules/admin/admin_reports.php
Callback monitor/modules/admin/callback_monitor.php
Nhật ký API/modules/admin/api_logs.php
Gán đơn/modules/admin/assign_order_user.php
Cách dùng nhanh
  • Realtime dashboard — danh sách đơn và trạng thái.
  • Chi tiết đơn — timeline, shipper, ảnh, log (theo dữ liệu đã ghi).
  • Callback monitor & API logs — xử lý lỗi tích hợp.
  • Users / Roles — tài khoản và phân quyền.

Chi tiết request/response: API.md. Xác thực thường qua header X-API-KEY (xem config/).

8. Trạng thái đơn (tham khảo)

Theo readme.md, chuỗi có thể gồm: new_order → assigned_pickup → picked_up → arrived_hub / in_transit → assigned_delivery → delivered (và failed khi lỗi).

Đối chiếu thực tế với bảng emslss_orders và emslss_tracking trong DB_SCHEMA.md hoặc install.sql.
